## How each file is structured

Every Q&A follows the same pattern for easy recall:

- **Memory Trick** – 3-5 word mnemonic to anchor the answer
- **4-5 bullet structure** – Each bullet is one key idea, 1-2 sentences
- **Real examples** – Drawn from your actual experience (Optum, Oracle Cerner, ML upgrade, OpenShift→K8s migration)
- **Strong closing line** – On selected questions, a quotable one-liner for impact

## Tips for the interview

- **Lead with the memory trick mentally** — it keeps your answer structured under pressure
- **Don't recite all 5 bullets** — use 2-3 in your spoken answer, hold the rest as backup if probed
- **Anchor in real examples** — interviewers remember stories, not generic frameworks
- **For technical questions** — answer at manager-level (architecture, trade-offs, team practices), not IC-level (line-by-line code)
- **Be honest about uncertainty** — "I'd lean X but the right answer depends on Y" is stronger than overclaiming

Good luck!
