
# Section 5: Production & Reliability

## Q1: How do you ensure production stability

- Monitoring & Visibility  
  - Use dashboards and alerts to detect issues early  

- Proactive Checks  
  - Regularly review API health, ETL pipelines, and system metrics  

- Risk Prevention  
  - Address known risks like vulnerabilities and platform upgrades proactively  

- Feedback Loop  
  - Use production incidents and metrics to improve system  

- Outcome  
  - Prevent issues before they impact customers  

---

## Q2: How do you handle production incidents / service down

- Phase 1: Immediate Response  
  - Detect via alerts or SLO breach  
  - Activate on-call and war room  
  - Communicate impact to stakeholders  

- Phase 2: Stabilization  
  - Check recent deployments and rollback if needed  
  - Identify if issue is application, infrastructure, or dependency  
  - Apply mitigation like throttling or graceful degradation  

- Phase 3: Resolution  
  - Fix root cause and deploy solution  
  - Validate system stability  

- Phase 4: Post-Incident  
  - Conduct RCA and define corrective actions  
  - Improve resilience and monitoring  

- Focus  
  - Minimize customer impact first, then ensure long-term stability  

---

## Q3: How do you manage support, on-call, and recurring issues

- Ownership  
  - Define clear on-call responsibilities within the team  

- Incident Handling  
  - Handle client issues, ETL failures, and API issues based on priority  

- Pattern Identification  
  - Identify recurring issues through trends  

- Permanent Fix  
  - Implement root cause fixes instead of temporary patches  

- Outcome  
  - Improves system stability and reduces repeated incidents  

---

## Q4: How do you ensure reliability during changes and releases

- Planning  
  - Align changes with release governance and approvals  

- Validation  
  - Ensure proper testing and readiness before deployment  

- Controlled Rollout  
  - Deploy changes in a controlled manner  

- Monitoring  
  - Closely monitor system post-deployment  

- Outcome  
  - Ensures safe and stable production changes  

---

## Q5: How do you manage platform updates and security risks

- Tracking  
  - Monitor platform updates, upgrades, and vulnerabilities  

- Prioritization  
  - Prioritize critical security and compliance requirements  

- Coordination  
  - Work with platform and architecture teams  

- Execution  
  - Plan updates without impacting delivery  

- Outcome  
  - Maintains system security and compliance  

---

## Q6: How do you drive operational excellence

- Governance  
  - Follow structured processes like CAB and release management  

- Reviews  
  - Participate in ops reviews, audits, and quality checks  

- Improvement  
  - Continuously improve based on incidents and feedback  

- Outcome  
  - Ensures high reliability and operational maturity
